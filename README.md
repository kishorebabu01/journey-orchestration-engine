# LLM-Powered Customer Journey Orchestration Engine

> **Portfolio Project 5 of 10** — Built by Kishore Babu as part of AI Growth Marketing portfolio.

---

## 🎯 What This System Does

A production-grade customer journey orchestration engine that:

1. **Captures** real-time user behaviour events via PostHog
2. **Classifies** every user into 1 of 7 journey states using a Python state machine
3. **Detects** journey triggers (state transitions + inactivity signals)
4. **Generates** deeply personalised messages using LLaMA 3.3 70B (free via Groq)
5. **Retrieves** relevant context from a RAG knowledge base before every message
6. **Delivers** messages via email (Brevo), push (OneSignal), and in-app (PostHog flags)
7. **Learns** from message outcomes — self-improving system that gets smarter weekly

Built as a portfolio demo using **Focusly** (fictional student productivity app) with simulated user events. All infrastructure, AI logic, and delivery systems are fully production-grade.

## 🧠 The 7 Journey States

| State | Description | Key Trigger |
|---|---|---|
| `NEW_SIGNUP` | Just signed up, no activity | activation_nudge_24h |
| `ACTIVATING` | Using features, not fully set up | progress_encouragement |
| `ACTIVATED` | Completed all onboarding steps | celebration_message |
| `RETAINED` | Regular active user | weekly_progress_insight |
| `CHURN_RISK` | Was active, gone quiet 7+ days | winback_day7 |
| `CHURNED` | Inactive for 30+ days | reactivation_monthly |
| `EXPANDING` | Power user, engagement score 80+ | upgrade_prompt |

## 🤖 AI Message Generation

Every message is generated by **LLaMA 3.3 70B** (via Groq API) with full RAG context:

```json
{
  "push_notification": "David, your 20-day streak is on fire 🔥",
  "email_subject": "Next Level Productivity",
  "email_body": "Hey David, you've completed 2,400 focus minutes...",
  "in_app_tooltip": "Unlock AI Coach — you've earned it",
  "channel_recommendation": "push",
  "reasoning": "High engagement score (87) and expanding state. Push chosen for immediacy. Achievement framing used based on Q2 campaign data showing 34% CTR for this segment."
}
```

Every AI decision is **fully logged with reasoning** in the `agent_decisions` table — explainable AI, not a black box.

| Table | Purpose |
|---|---|
| `users` | User profiles + current journey state |

| `journey_triggers` | Pending + processed trigger queue |
| `agent_decisions` | Complete AI reasoning audit trail |
| `messages_sent` | Delivery log for all channels |
| `message_outcomes` | Engagement tracking (opens, clicks) |
| `rag_documents` | Vector knowledge base (384-dim embeddings) |

---

## 📚 RAG Knowledge Base

The AI agent retrieves context before every message from a **45-chunk knowledge base** indexed in Supabase pgvector:

- `past_campaigns.csv` — 12 historical campaign performance records
- `message_performance_history.csv` — 12 message performance records by segment
- `brand_voice.md` — 10 brand voice guideline sections
- `product_features.md` — 5 feature documentation sections
- `user_segments.md` — 6 user segment insights

Embedding model: `sentence-transformers/all-MiniLM-L6-v2` (free, runs locally)

## 🏃 Running Locally

```bash
# Clone repository
git clone https://github.com/kishorebabu01/journey-orchestration-engine.git
cd journey-orchestration-engine

# Create virtual environment
python -m venv venv
venv\Scripts\activate  # Windows

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Add your API keys to .env

# Run simulated users
python data/simulate_events.py

# Run state machine
python -m state_machine.evaluator

# Run AI agent
python -m agent.journey_agent

# Run delivery layer
python -m delivery.n8n_webhook
```
